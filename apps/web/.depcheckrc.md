# Depcheck Configuration

This configuration helps depcheck understand our Next.js/Tailwind CSS v4 setup.

## Ignored Packages

Packages listed in the `ignores` array are legitimately used but can't be detected by static analysis:

### Build Tool Dependencies
- `@types/*` - TypeScript type definitions (used by compiler, not imported)
- `eslint-*` - ESLint plugins (loaded via config file)
- `prettier` - Code formatter (standalone tool)
- `typescript` - TypeScript compiler (build tool)
- `vitest` - Test runner (standalone tool)
- `playwright` - E2E testing framework (standalone tool)

### Framework Dependencies (Config File Usage)

#### @copilotkit/runtime
- **Used in:** `next.config.ts`
- **How:** Listed in `experimental.optimizePackageImports` array
- **Why needed:** Next.js requires this package for bundle optimization even though it's not directly imported in source code

#### @tailwindcss/postcss
- **Used in:** `postcss.config.mjs`
- **How:** Referenced as a PostCSS plugin: `plugins: { "@tailwindcss/postcss": {} }`
- **Why needed:** PostCSS build pipeline requires this plugin at build time

#### @tailwindcss/typography
- **Used in:** Throughout the app via CSS classes
- **How:** Provides `prose`, `prose-sm`, `prose-invert`, etc. utility classes
- **Why needed:** Used in `MarkdownRenderer.tsx` and other components for markdown styling
- **Detection issue:** Depcheck can't analyze CSS class usage in JSX

#### tailwindcss
- **Used in:** `src/app/globals.css`
- **How:** Imported via `@import "tailwindcss";` directive (Tailwind CSS v4 syntax)
- **Why needed:** Core CSS framework that processes the CSS at build time
- **Detection issue:** Depcheck can't analyze CSS @import directives

## Running Depcheck Locally

```bash
cd apps/web
npx depcheck
```

The configuration in `.depcheckrc` will be automatically loaded.

## CI/CD

The GitHub Actions workflow in `.github/actions/scan-javascript-dependencies/action.yml` uses this same configuration file, ensuring consistency between local development and CI.
